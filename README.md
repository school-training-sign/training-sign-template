# 학교 연수 전자서명 배포 템플릿

다른 학교가 자기 Google 계정과 GitHub 계정으로 독립 운영할 수 있도록 만든 빈 배포본입니다. 이 저장소에는 특정 학교의 명단·연수·서명·비밀번호·공유 키·Apps Script 배포 주소가 들어 있지 않습니다.

이 시스템은 교직원 연수 참여 확인용입니다. 본인 인증이 필요한 공식 동의·결재·법적 전자서명 용도로 사용하지 마세요.

## 가장 쉬운 설치 순서

1. 전달받은 Google 시트 배포용 원본에서 `파일 → 사본 만들기`를 누릅니다.
2. 학교용 사본에서 `🖊️ 전자서명 관리 → 초기 설정 실행`을 누르고 권한을 승인합니다.
3. `확장 프로그램 → Apps Script → 배포 → 새 배포 → 웹 앱`으로 이동합니다.
4. 실행 계정은 `나`, 액세스 권한은 `모든 사용자`로 설정하고 `/exec` 주소를 복사합니다.
5. GitHub에서 이 저장소의 `Use this template → Create a new repository`를 누릅니다.
6. 새 저장소의 `assets/config.js`에 있는 `__APPS_SCRIPT_WEB_APP_URL__`을 복사한 `/exec` 주소로 바꿉니다.
7. `Settings → Pages`에서 `Deploy from a branch`, `main`, `/(root)`를 선택합니다.
8. 만들어진 GitHub Pages 주소에서 관리자 첫 설정을 진행합니다.

Google 시트 배포용 원본을 받지 못했다면 저장소의 [학교 연수 전자서명 시스템 - 배포용 원본.xlsx](학교%20연수%20전자서명%20시스템%20-%20배포용%20원본.xlsx)를 Google Drive에 올려 Google 시트로 변환한 뒤, `apps-script/Code.gs`와 `apps-script/appsscript.json`을 연결형 Apps Script에 넣어야 합니다.

자세한 설치·운영·복구 방법은 [OPERATIONS.md](OPERATIONS.md)에 정리되어 있습니다.

## 주요 기능

- 연수 선택 → 본인 선택 → 터치·마우스 서명의 3단계 참여 화면
- 특정 날짜 또는 매일 연수와 운영 시간 설정
- 구성원 일괄 입력, 엑셀·CSV 가져오기, 부서명 변경
- 공유 링크·QR과 공유 키 교체
- 실제 서명이 들어간 A4 미리보기, PDF·엑셀·인쇄
- 비공개 Google Drive 원본 저장과 보관 완료 후 원본 삭제
- 숫자 4자리 또는 문자·숫자 조합 관리자 비밀번호

## 개인정보와 보안

- 공유 링크에는 별도 본인 인증이 없습니다. 학교 내부에서만 안내하세요.
- 관리자용 Google 시트는 관리용 계정 외에 편집 권한을 주지 마세요.
- 공개 GitHub 저장소에는 초기 설정 코드·비밀번호·공유 키·명단을 넣지 마세요.
- PDF 또는 엑셀 보관을 확인한 뒤 시스템 원본을 삭제하세요. 인쇄 작업만으로는 원본 삭제가 허용되지 않습니다.
- 학교가 정한 보관기간이 지나면 비공개 Drive의 출력 파일도 삭제하세요.

별도 오픈소스 라이선스는 부여하지 않았습니다. 포함된 제3자 라이브러리는 [THIRD_PARTY_NOTICES.md](THIRD_PARTY_NOTICES.md)의 각 라이선스를 따릅니다.
