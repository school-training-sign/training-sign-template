# 설치 및 운영 설명서

## 1. 준비

- 학교가 교직원 명단과 서명을 저장하도록 승인한 관리용 Google 계정과 2단계 인증
- GitHub 계정, Git, GitHub CLI
- 관리자들이 함께 사용할 숫자 4자리 또는 문자·숫자 조합 10자 이상의 비밀번호
- 학교가 정한 개인정보 수집 목적, 수집 항목, 보관기간·삭제 방법

공유 링크 방식은 별도 본인 인증이 없습니다. 링크를 받은 사람은 명단을 보고 다른 사람 이름으로 제출할 수 있으므로 학교 내부에서만 공유해야 합니다.

## 2. 비공개 배포용 원본 만들기

1. 저장소에서 제공하는 `학교 연수 전자서명 시스템 - 배포용 원본.xlsx`를 관리용 Google Drive에 업로드하고 Google 스프레드시트로 변환합니다.
2. 파일 이름을 `학교 연수 전자서명 시스템 - 배포용 원본`으로 정합니다.
3. `확장 프로그램 → Apps Script`를 열고 `apps-script/Code.gs`와 `apps-script/appsscript.json`을 연결합니다.
4. Apps Script 프로젝트 이름을 `학교 연수 전자서명 시스템`으로 정합니다.
5. 프로젝트 설정의 스크립트 속성에 `TEMPLATE_LOCK`=`1`을 추가합니다.
6. 명단·연수·서명·관리자 비밀번호·공유 키·웹앱 배포 URL을 넣지 않은 채 비공개로 보관합니다.

배포용 원본에서 초기 설정을 누르면 잠금 안내가 표시되는 것이 정상입니다. 원본 시스템의 시트 사본과 코드는 참고용으로만 두고 수정하지 않습니다.

## 3. 학교용 사본 초기화

1. 배포용 원본에서 `파일 → 사본 만들기`로 실제 학교용 사본을 만듭니다. Google 시트를 복사하면 연결된 Apps Script도 새 프로젝트로 함께 복사됩니다.
2. 학교용 사본을 새로 열어 `🖊️ 전자서명 관리 → 초기 설정 실행`을 선택합니다.
3. Drive·Sheets·트리거 권한을 승인합니다. 별도 데이터 스프레드시트는 만들지 않고 현재 학교용 사본에 `설정`, `구성원`, `연수`, `서명`, `출력 작업`, `감사 기록` 탭을 구성합니다.
4. 실행이 끝나면 데이터 탭은 숨겨지고 `사용설명서`만 표시됩니다. 필요할 때 관리 메뉴에서 데이터 탭을 다시 표시할 수 있습니다.
5. 대화상자에 표시된 초기 설정 코드를 안전한 곳에 잠시 적어 둡니다. 첫 비밀번호 설정 후 자동 폐기됩니다.

처음 권한을 승인할 때 `Google에서 확인하지 않은 앱`이 나오면 개발자 이메일이 지금 사본을 만든 본인 계정인지 확인하세요. 본인 계정이 맞을 때만 `고급 → 학교 연수 전자서명 시스템으로 이동 → 허용`으로 진행합니다. 모르는 이메일이 표시되거나 조직 정책으로 차단되면 진행하지 말고 Google Workspace 관리자에게 문의하세요. 이 안내는 소유자의 최초 승인에만 표시되며, 일반 서명 참여자에게는 나타나지 않습니다.

초기화를 다시 실행해도 기존 데이터·비밀번호·공유 키는 지우지 않습니다. 누락된 탭·헤더·비공개 폴더·정리 트리거만 복구합니다.

## 4. 학교용 웹앱 배포

1. 학교용 사본에서 `확장 프로그램 → Apps Script`를 엽니다.
2. `배포 → 새 배포 → 웹 앱`을 선택합니다.
3. 실행 계정은 **나**, 액세스 권한은 **모든 사용자**로 정합니다.
4. 배포하고 `/exec`로 끝나는 웹앱 URL을 복사합니다. 시트의 `🖊️ 전자서명 관리 → 웹앱 주소 확인`에서도 확인할 수 있습니다.
5. 코드가 바뀌면 `배포 관리 → 수정 → 새 버전`으로 다시 배포합니다. 기존 URL은 유지됩니다.

웹앱 실행 중에는 활성 시트를 사용하지 않습니다. 초기화 때 Script Properties에 저장한 학교용 시트 ID로만 데이터를 엽니다.

## 5. GitHub Pages 연결

1. `assets/config.js`의 `__APPS_SCRIPT_WEB_APP_URL__`을 복사한 `/exec` URL로 바꿉니다.
2. GitHub에 공개 저장소 `training-sign`을 만들고 이 폴더의 파일을 `main` 브랜치에 올립니다.
3. 저장소 `Settings → Pages`에서 `Deploy from a branch`, `main`, `/ (root)`를 선택합니다.
4. `https://사용자명.github.io/training-sign/` 주소가 열리는지 확인합니다.

공개 저장소에는 Apps Script 배포 URL만 들어갑니다. 초기 설정 코드, 관리자 비밀번호, 공유 키, 명단, 서명 파일 ID는 저장소에 넣지 마세요.

## 6. 관리자 첫 설정

1. GitHub Pages 기본 주소에서 `관리자`를 누릅니다.
2. 초기 설정 코드와 숫자 4자리 또는 문자·숫자 조합 10자 이상의 관리자 비밀번호를 입력합니다.
3. 기관 설정에서 학교명, 부제목과 개인정보 안내 항목을 모두 입력합니다.
4. 구성원 명단과 연수를 등록합니다. 연수마다 전체 구성원 또는 서명 대상 부서를 선택할 수 있고, 필요할 때만 `현장 접수 코드 사용`을 켤 수 있습니다. 기존 연수와 이 항목을 끈 연수는 코드 없이 기존 방식으로 운영됩니다. 개인정보 안내가 비어 있으면 연수를 활성화할 수 없습니다.
5. 공유 메뉴에서 링크와 QR을 복사해 교직원에게 안내합니다.

## 7. 연수 운영

1. 연수 날짜와 가능 시간을 확인하고 활성화합니다.
   - 지난 날짜의 고정 연수도 활성화하면 다시 서명을 받을 수 있습니다. 연수일은 원래 날짜로 표시하고 실제 제출 날짜·시각은 별도로 기록합니다.
   - 지난 고정 연수의 시작·종료 시각은 연수 당일 일정으로 보며, 재수합 중에는 활성 상태가 접수 여부를 결정합니다. 미래 고정 연수는 해당 날짜 전까지 표시되지 않습니다.
2. QR이나 공유 링크를 학교 내부 메신저·화면으로 안내합니다.
3. 현장 접수 코드를 사용하는 연수는 연수 카드의 `현장 접수`를 열고 5·10·15분 중 접수 시간을 고른 뒤 `접수 시작`을 누릅니다. 화면에 한 번 표시되는 6자리 코드를 현장에서만 안내합니다.
   - 새 코드를 발급하면 이전 코드는 즉시 무효가 되고, `접수 종료`를 누르거나 선택 시간이 지나면 제출할 수 없습니다.
   - 오늘 또는 매일 연수는 등록된 시작·종료 시각 안에서만 접수를 열 수 있으며, 선택 시간이 남아 있어도 연수 종료 시각에 맞춰 자동 종료됩니다. 과거 고정 연수의 재수합은 기존처럼 시각 제한을 적용하지 않습니다.
   - 관리자 화면을 닫은 뒤에는 기존 코드를 다시 볼 수 없습니다. 접수가 계속 열려 있더라도 새 코드를 발급해 다시 안내해야 합니다.
   - 매일 연수의 코드는 서버 기준 오늘 날짜에만 유효합니다. 과거 연수라도 코드 기능을 켠 경우에는 관리자가 현재 접수를 열어야 제출할 수 있습니다.
4. 연수 목록의 `미서명 현황`에서 해당 연수의 대상 부서에 속한 미서명자를 확인합니다. `전체 현황`으로 바꾸면 완료 시각까지 볼 수 있습니다.
5. 서명 기록 메뉴에서 `현장 코드 확인·일반 링크·기존 기록` 표시를 확인하고, 잘못된 기록을 개별 삭제합니다.
6. 연수 목록에서 해당 연수의 `출력`을 펼쳐 날짜·등록순/부서순/이름순·열 수와 PDF/엑셀/인쇄를 선택합니다.
7. 실제 서명이 포함된 A4 미리보기를 확인한 뒤 선택한 파일을 만들거나 인쇄합니다.
8. PDF 또는 엑셀 파일을 학교가 정한 위치에 보관한 뒤 `원본 삭제`에서 연수명을 다시 입력합니다. 인쇄 작업만으로는 원본 삭제가 열리지 않습니다.

출력 파일은 비공개 드라이브의 `출력 보관` 폴더에 남습니다. 학교의 보존기간이 끝나면 관리용 Google 계정에서도 직접 삭제하세요.

## 8. 보안·복구

- 링크 유출: 관리자 `공유·보안 → 공유 키 교체` 후 새 QR과 링크를 배포합니다.
- 현장 코드 노출·입력 잠금: 연수 카드의 `현장 접수`에서 `새 코드 발급`을 눌러 이전 코드와 해당 접수의 오입력 잠금을 함께 초기화합니다. 코드 추측을 막기 위해 구성원별·접수 전체 오입력 횟수를 제한합니다.
- 관리자 비밀번호 분실: 학교용 시트에서 `🖊️ 전자서명 관리 → 관리자 비밀번호 복구`를 선택합니다. 기존 관리자 세션은 모두 무효화됩니다.
- 데이터 탭 점검: 관리 메뉴의 `데이터 탭 표시·숨기기`를 사용합니다. 관리용 Google 계정 외에는 시트 편집 권한을 주지 않습니다.
- 관리 계정 변경: 시트와 비공개 폴더의 소유권을 학교가 승인한 새 계정으로 이전한 뒤 새 계정에서 웹앱을 재배포합니다.
- 출력 중단: 해당 연수의 `출력` 영역에서 `다시 시도`를 누릅니다. 미리보기까지 준비된 작업은 `미리보기`로 다시 열 수 있습니다. 24시간이 지난 미완료 작업과 임시 미리보기는 자동 정리됩니다.

관리자 비밀번호를 여러 명이 공유하므로 누가 어떤 관리 작업을 했는지는 구분되지 않습니다. 감사 기록에는 작업 종류·시각·건수만 남고 교직원 이름은 불필요하게 복제하지 않습니다.
